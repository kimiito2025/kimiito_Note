# コレクション利用手順

対象ファイル:
- 03_Postman_Collection_Import後Calling適用.json

## 1. インポート
- Postman で Import を開き、上記 JSON を取り込む。

## 2. 先に設定する変数
- access_token
- target_email
- location_id
- license_id
- extension
- hunt_group_name
- pickup_group_name

## 3. 実行順
1. 0) ユーザ検索（email -> person_id）
2. 1) ユーザ詳細確認（callingData=true）
3. 2) 内線適用（People PUT, callingData=true）
4. 3) ハントグループ一覧取得（URL変数）
5. 4) ハントグループ詳細取得（URL変数）
6. 5) ハントグループ更新（URL変数, body要編集）
7. 6) ピックアップグループ一覧取得（URL変数）
8. 7) ピックアップグループ詳細取得（URL変数）
9. 8) ピックアップグループ更新（URL変数, body要編集）

## 4. 重要ポイント
- People PUT は全体更新性があるため、1) の戻り値を見て未指定項目が消えないように body を補完して実行する。
- Hunt/Pickup は API バージョン差分で URL や body スキーマが異なる場合がある。
- そのため本コレクションでは URL を変数化している。自テナントの最新ドキュメントに合わせて以下を必要に応じて修正する。
  - hunt_group_list_url
  - hunt_group_get_url
  - hunt_group_update_url
  - pickup_group_list_url
  - pickup_group_get_url
  - pickup_group_update_url

## 5. エラー時の確認
- 400/409 が出たら、対象ユーザや対象グループを GET で再確認する。
- 403 はスコープ不足が多い。Calling 機能操作には telephony_config 系権限が必要。
- 429/5xx は再試行を前提にする。
