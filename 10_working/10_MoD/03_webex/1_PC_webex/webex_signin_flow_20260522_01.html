時系列で「最初にWebex Edge到達 → 次にMicrosoft認証 → 最後にWebex認証基盤とサービス初期化」を追える構成です。

各ステップの説明

番号 通信 何をしているか
① PC → DNS Webex アプリが最初に ty6-wxt-jp.bcld.webex.com の名前解決を行います。まずどのIPアドレスに接続すべきかをDNSに問い合わせている段階です。
② PC → Webex Edge (TLS :443) DNSで得たIPアドレス（135.84.169.185）に対して HTTPS/TLS セッションを確立します。Webex の日本リージョンEdge（BCLDゲートウェイ）への入口接続です。
③ PC → Webex Edge (TCP :8934) Webex Calling 用のシグナリングポート（8934）への接続を開始します。通話機能を利用するための専用チャネルです。
④ PC → Microsoft IdP (TLS :443) Microsoft の認証基盤（login.live.com / login.microsoftonline.com）へ TLS 接続します。Webex が Microsoft アカウント（Azure AD / Entra ID）でのサインインを求めている段階です。
⑤ Microsoft IdP → PC Microsoft から認証結果（IDトークン / アクセストークン）が返却されます。この応答に認証成功の証明が含まれます。
⑥ PC → Webex IdBroker (TLS :443) Microsoft から受け取ったトークンを Webex 側の ID ブローカー（idbroker-sg.webex.com）に提示します。Webex は自社のアカウントシステムと Microsoft アカウントを照合し、Webex セッションを発行します。
⑦ PC → Webex Identity (TLS :443) Webex Identity サービス（identity-sg.webex.com）に接続し、ユーザーのプロフィール・権限・ライセンス情報を取得します。サインインが実質的に完了するフェーズです。
⑧ PC → Webex Core Services (TLS :443) サインイン完了後、Webex アプリが各機能（デバイス管理・設定・連絡先・メッセージ・通話・会議・カレンダーなど）のAPIエンドポイントへ一斉に初期接続します。これによりホーム画面が表示される状態になります。
⑨ Webex Core Services → PC 各サービスから初期データ（連絡先一覧・未読メッセージ数・設定値など）が返却され、Webex アプリの表示が完成します。
